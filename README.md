**5 · Experiments Prompt**

```text
### Prompt ❻ 实验结果解析

0. **快速导航**  
   - ★ 请先扫描全文，定位所有实验相关的 **表格、图、附录** 页码，避免遗漏。  
   - 建议依次处理：数据集 → 主结果 → 消融 → 置信度 → 复现细节。

1. **数据集与划分 (Dataset Splits)**  
   - 列表格式：| Dataset | #Train | #Val | #Test | Task | 来源/链接(可选) |  
   - 若作者只给比例：计算出样本数或注明“n/a”。  
   - 多语言/多模态数据需在 Task 栏补充说明；跨数据集统一写多行。  
   - ★ 检查是否提到公开/私有数据、授权许可。

2. **关键结果汇总 (Main Results)**  
   - 对每张主表/主图：  
     1. 引用表号/图号 + 标题。  
     2. 提取 **最佳指标**（原值+单位），对比 baseline/SOTA。  
     3. 计算并报告：  
        - 绝对提升 = New – Baseline  
        - 相对百分比 = (New – Baseline) / Baseline × 100 %  
     4. 用列表 or 1 行表格展示：| Metric | Baseline | Ours | ΔAbs | ΔRel% |  
   - 若有多任务/多语言，逐任务独立统计。  
   - ★ 对曲线图：捕获峰值或 AUC，并注明读取方式（直接读值或估算）。  

3. **消融实验 (Ablation Studies)**  
   - 构建表：| 移除组件 | 指标变化 | ΔAbs | 作者结论 | 你的解读(≤1 行) |  
   - 若作者以条形图展示，需转录数值；估算时标 “(估算)”。  
   - ★ 指出作者是否分析交互效应 (two-way ablation)，若无可建议。

4. **统计显著性 & 置信区间 (Significance)**  
   - 若作者给 *p*-value / CI：  
     - 按表列 “指标 | 值 | p/CI | 是否显著(α=0.05)”  
   - 若缺失：输出警示 “⚠️ 未报告统计显著性，结果可能不稳健”，并简述风险 (如随机种子敏感)。  
   - 可建议使用 **bootstrap** 或 **t-test** 等方法补充。  

5. **复现要点 (Reproducibility Tips)**  
   - 列清单：  
     - **硬件**：GPU/TPU 型号 & 数量，CPU, RAM。  
     - **训练时长**：单 epoch / 总小时，Batch Size。  
     - **显存占用**：峰值 (GB)；若未给需估算或标 “n/a”。  
     - **随机种子**：是否固定；指明库 (PyTorch/TF) 与 seed 值。  
     - (可选) **代码链接/Commit**：若作者公开。  
   - ★ 若文中未提及某项，直接写 “未说明”，并用 “(潜在复现风险)” 标注。  
6. **一致性 & 补漏检查**  
   - 对照已有 Prompt：确认所有 **公式引用的指标** 与本节结果 **完全对应**。  
   - 若发现遗漏表/图/数值：  
     - 立即在对应小节补充，并在此列 “补漏 X 项：表 2, Fig 3 …”。  
   - 完成后输出总结：“实验解析完成，共覆盖 Y 张表 / Z 张图，补漏 K 项。”  
```
